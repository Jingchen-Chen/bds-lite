# Cluster-aware statistics

The manuscript's primary evidence is **cluster-aware**, computed by `analysis/generate_rescue_analysis.py`. This document describes the methods and the reproducibility caveat for the bootstrap confidence intervals.

## Unit of clustering

Per-case oriented deltas (BDS-Lite minus U-Net, averaged over the three seeds) are grouped by a **cluster id** so that statistically dependent slices are not treated as independent:

- **ISIC2018:** cluster = image (each 2D dermoscopy image is its own cluster).
- **ACDC:** cluster = patient (`patient\d+`); multiple slices/frames per patient share a cluster.
- **Synapse:** cluster = case/volume (`case\d+`); axial slices of one volume share a cluster.

Cluster counts of record: ISIC 519, ACDC 20, Synapse 12 (`n_clusters` in `analysis/outputs/cluster_level_statistics.csv`).

## Quantities computed per (dataset, metric)

For the cluster-mean oriented deltas:

1. **Point estimates** — mean and median oriented delta. Deterministic.
2. **Cluster bootstrap 95% CI** — resampling clusters with replacement, 5000 iterations, percentile interval. **Uses the shared RNG (see caveat below).**
3. **Wilcoxon signed-rank test** (`scipy.stats.wilcoxon`) on the cluster means. Deterministic.
4. **Holm correction** across the metric family, per dataset. Deterministic.
5. **Sign-based effect size** — `(#clusters improved − #clusters worsened) / #nonzero`. Deterministic.
6. **Leave-one-cluster-out (LOCO)** — recompute the mean dropping one cluster at a time; report the min and max. Deterministic.

The decision column (`decision_alpha_0_05`) uses the Holm-adjusted p at α = 0.05.

## Bootstrap-RNG reproducibility caveat

The script seeds a **single module-global** NumPy generator, `RNG = np.random.default_rng(20260606)`, and consumes it in script order. As a consequence:

- The **bootstrap CI endpoints** (`bootstrap_ci_low`, `bootstrap_ci_high`) reproduce **exactly only when the full analysis script is run end-to-end** — i.e. via `reproduce.sh` or by invoking `analysis/generate_rescue_analysis.py` directly. They will **not** reproduce if functions are called out of order or in isolation (e.g. from a notebook), because the RNG draw sequence would differ.
- **Every non-bootstrap quantity** above (point estimates, Wilcoxon statistics, Holm-adjusted p-values, sign effect sizes, LOCO ranges) is fully deterministic and reproduces exactly regardless of call order.

The committed `analysis/outputs/cluster_level_statistics.csv` is the **artifact of record** and is **not** regenerated by this repository's build. The reproduction was verified by re-running the script to a scratch directory and diffing (see `analysis/README.md` and `docs/PROVENANCE` notes); deterministic columns matched exactly and the bootstrap endpoints matched within the caveat above.

The pinned `numpy==2.4.4` / `scipy==1.17.1` versions (see `requirements.txt`) are part of this guarantee: a different NumPy generator implementation can change the bootstrap draw stream even with the same seed.
