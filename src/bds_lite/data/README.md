# Data preprocessing provenance

This package restores the data-preprocessing source from the author's
source checkout (the original `src/bds_lite/data/` tree). The source files
restored byte-for-byte are:

- `boundary.py`
- `converters.py`
- `datasets.py`
- `schema.py`

`__init__.py` is restored with one release-only compatibility alias so
`import bds_lite.data.mask_to_sdf` resolves to the restored `boundary.py`
implementation while preserving the original top-level
`from bds_lite.data import mask_to_sdf` function. The SDF implementation itself
is not duplicated.

## Boundary targets

`mask_to_boundary` converts a 2D integer label mask into a binary boundary
target. For each foreground label, excluding background label 0 and any
`ignore_index`, it computes:

1. `scipy.ndimage.binary_dilation(region, iterations=radius)`
2. `scipy.ndimage.binary_erosion(region, iterations=radius)`
3. the logical XOR of those two masks

The per-class boundaries are OR-combined into one binary `uint8` map. With the
default `radius=1`, this is the documented 3 by 3 morphological-gradient target
used by the training-time boundary decoder. `mask_to_one_hot_boundary` applies
the same rule per foreground class and stacks the results as `[K, H, W]`.

## Signed-distance targets

`mask_to_sdf` returns one signed-distance map per foreground class. Distances
come from SciPy's exact Euclidean distance transform:

- negative values are inside the foreground class;
- positive values are outside the foreground class;
- an absent class returns an all-zero map;
- an empty binary mask returns all zeros;
- an all-foreground binary mask returns `-1` everywhere.

The stored SDF targets are `float32` and are not clipped or normalized during
target generation. The training losses normalize distances internally where
needed.

## Dataset conversion and loading

The converters write compressed `.npz` samples with `image`, `mask`, and
`sample_id` arrays. The loader reads samples from `root/<split>/*.npz` and can
require optional `boundary` and `sdf` arrays generated by
`scripts/prepare_boundary_targets.py`.

Preprocessing behavior restored from source:

- ISIC2018: RGB images are scaled to `[0, 1]`, transposed to CHW, and resized
  with bilinear interpolation when requested; binary masks use nearest-neighbor
  interpolation.
- ACDC official NIfTI: volumes are min-max normalized per volume before 2D
  slicing; masks use nearest-neighbor interpolation.
- ACDC TransUNet H5: pre-sliced `image` arrays are preserved as provided
  (the source format is documented as z-score normalized); masks use
  nearest-neighbor interpolation when resized.
- Synapse HiFormer layout: pre-sliced training images and H5 test-volume
  slices are preserved as provided; images use bilinear interpolation and masks
  use nearest-neighbor interpolation when resized.

The restored loader does not read manifest JSON files directly. The committed
split manifests document and audit the locked partitions; the processed data
must be staged into split directories matching those manifests for exact
reproduction.
